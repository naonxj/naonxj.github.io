CTF Challenge Design: The Ancient Tomb

팀별 OVA 가상 머신 제작 및 취약점 시나리오 설계

OVA Deployment Web Hacking Steganography RCE Exploit

01. Project Overview

본 프로젝트는 팀별로 직접 취약점이 포함된 시스템을 설계하여 OVA 파일로 배포하고, 타 팀이 해당 시스템에 침투하여 Flag를 획득하는 해킹 대회를 목적으로 진행되었습니다.

출제 취약점: Unrestricted File Upload

제가 기획하고 구현한 메인 취약점은 파일 업로드 취약점입니다. 사용자가 접근하는 웹 페이지 UI 상에서는 '권한 없음' 메시지를 띄워 업로드를 차단하지만, 실제 서버 측 처리 파일인 upload.php에서 권한 검증 로직이 누락된 점을 이용하여 서버의 제어권을 탈취하는 시나리오를 구성했습니다.

02. Exploit Walkthrough

공격자가 시스템의 허점을 찾아내고 최종적으로 RCE(Remote Code Execution)를 달성하는 과정입니다.

1 / 5
[Step 1] 관리 페이지 및 UI 제한 확인
Tomb.php 접속 시 "No upload permission"이라는 문구가 출력됩니다. 시각적으로는 업로드 기능을 사용할 수 없도록 설계하여 공격자의 혼란을 유도했습니다.
2 / 5
[Step 2] 정보 수집 및 숨겨진 경로 탐색
dirb 도구를 활용해 서버를 스캔한 결과, 업로드된 파일들이 저장되는 /uploads 디렉토리를 발견하여 공격 지점을 식별했습니다.
3 / 5
[Step 3] 공격용 웹쉘(test.php) 제작
서버 내부 정보를 확인하고 Flag를 출력하기 위해 vi test.php로 시스템 명령어를 포함한 PHP 페이로드를 직접 작성했습니다.
4 / 5
[Step 4] 인터페이스 우회를 통한 파일 업로드
UI를 거치지 않고 curl 명령어로 upload.php에 직접 파일을 전송했습니다. 서버 단의 권한 체크 부재로 인해 shell.php가 성공적으로 업로드되었습니다.
5 / 5
[Step 5] 원격 코드 실행 및 최종 Flag 탈취
/uploads/shell.php에 접근하여 코드를 실행했습니다. 서버 내부 명령어가 수행되어 화면에 최종 Flag가 출력됨으로써 시스템 장악에 성공했습니다.
