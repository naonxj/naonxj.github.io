00. Network Topology

전체 네트워크 구성도 (GNS3)

GNS3를 이용하여 가상 네트워크 인프라를 구축하고, 방화벽 정책 설정, 라우터 이중화(HSRP), VPN 서버 구축, IDS 및 SIEM 시스템을 각각 개별적으로 구축하여 실습한 프로젝트입니다.

01. GNS3 Infrastructure Setup

GNS3 내 가상 장비를 활용하여 방화벽 정책과 라우터 이중화(HSRP)를 구성했습니다.

1. Firewall Rules (Cisco ASAv)

Rule 1: Firefox3에서 Inside 구역으로 http 접속 허용
Rule 1: Firefox3에서 Inside 구역으로 telnet 접속 허용
Rule 2: Webterm2에서 dmz 구역으로 telnet 접속 허용
Rule 3: PC1에서 Webterm1으로 ping 통신 허용
Rule 4: Webterm1에서 R1 라우터로 http 접속 허용
Rule 4: Webterm1에서 R1 라우터로 telnet 접속 허용
Rule 5: Webterm2에서 PC1으로 ping 통신 허용

2. HSRP Configuration (High Availability)

라우터 장애에 대비하여 HSRP(Host Standby Router Protocol)를 설정했습니다.

Active router: 주 경로로 동작하는 라우터 설정 (R4), Standby router: 대기 경로로 동작하는 라우터 설정 (R3) , Virtual Gateway : 192.168.50.254 설정

R4: HSRP Active 상태 확인 (Primary Gateway)
R3: HSRP Standby 상태 확인 (Failover 대기)

02. VirtualBox Security Solutions

VirtualBox에 설치된 가상 머신들을 연동하여 VPN, IDS, SIEM 보안 서버를 구축했습니다.

1. OpenVPN Server

VirtualBox의 pfSense를 활용해 OpenVPN 서버를 구축했습니다. 외부(윈도우 PC) 클라이언트가 가상 터널(10.10.10.0/24)을 통해 내부 네트워크(192.168.16.0/24)에 안전하게 원격 접속할 수 있는 환경을 구현했습니다.

OpenVPN 서버 구축 후 외부 Win10 클라이언트 접속 확인
가상터널(10.10.10.0/24) 설정 및 내부 네트워크(192.168.16.0/24) 설정

3. Threat Detection Scenarios (Snort)

커스텀 Snort Rule을 활용한 5가지 위협 시나리오별 공격(Attempt)과 탐지(Detection) 결과입니다.

Scenario 1: ICMP & HTTP

[공격] Ping 시도
[탐지] ICMP 알람
[탐지] HTTP 트래픽 식별

Scenario 2: Inter-Network Ping

[공격] Rocky9 -> Win10
[탐지] 네트워크 간 핑 탐지

Scenario 3: Command Injection

[공격] /etc/passwd 접근
[탐지] 인젝션 패턴 매칭

Scenario 4: DDoS (Land Attack)

[공격] Random IP DDoS
[탐지] 트래픽 플러딩 탐지

Scenario 5: SYN Scanning

[공격] 포트 스캐닝 시도
[탐지] SYN 스캔 행위 식별

4. Threat Detection Scenarios (Suricata)

Suricata IDS를 활용하여 동일한 5가지 시나리오에 대한 탐지 성능을 교차 검증했습니다.

Scenario 1: ICMP & HTTP

[공격] Win10 통신 시도
[탐지] fast.log 경보
[탐지] HTTP 시그니처 탐지

Scenario 2: Inter-Network Ping

[공격] Rocky9 호스트발 통신
[탐지] 실시간 알람 수신

Scenario 3: Command Injection

[공격] 악성 페이로드 전송
[탐지] 명령어 삽입 식별

Scenario 4: DDoS (Land Attack)

[공격] 위조 패킷 플러딩
[탐지] 임계치 기반 탐지

Scenario 5: SYN Scanning

[공격] Nmap 포트 스캔
[탐지] 스캔 패턴 감지

4. OSSEC 구축

OSSEC 매니저 서버에서 생성한 인증 키를 Windows 10(192.168.16.131) 및 Rocky Linux(192.168.16.170) 에이전트에 각각 등록하고, 서버-클라이언트 간의 정상적인 연결 상태를 확인하여 시스템 구축을 완료했습니다.

1-1. [server] Windows 10(192.168.16.131) agent 추가 및 인증 키 생성
1-2. [server] Windows 10(192.168.16.131) agent 등록 확인 및 인증 키 추출
1-3. [Agent] Windows Agent Manager에서 서버 ip 및 인증 키 등록
2-1. [server] - Rocky(192.168.16.170) agent 추가 및 인증 키 생성
2-2. [server] - Rocky(192.168.16.170) agent 인증 키 임포트 및 연결
OSSEC 실시간 경보(Alert) 로그 모니터링 및 정상 작동 확인

5. Zabbix 구축

Rocky Linux 기반의 Zabbix Server를 구축하고, Windows 10 및 Ubuntu 호스트를 에이전트로 등록하여 시스템 자원 및 네트워크 가동 상태를 실시간으로 모니터링하는 환경을 조성했습니다.

1. Zabbix Server 메인 대시 보드 : 전체 시스템 현황 시각화
2. 호스트 구성 정보: Windows 10 및 Ubuntu 에이전트의 정상 연결(ZBX 활성화) 확인
