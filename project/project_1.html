04. Threat Detection Scenarios (Snort Rules)

커스텀 Snort Rule을 작성하여 5가지 유형의 공격을 시뮬레이션하고, 패킷 기반 탐지 로그를 확인했습니다.

Scenario 1: ICMP & HTTP Traffic Detection

Win10 -> OSSEC (Ping 시도)
Snort 탐지 로그 (ICMP)
HTTP 트래픽 탐지 로그 확인

Scenario 2: Inter-Network Ping (Rocky -> Win10)

공격: Rocky9에서 Ping 전송
탐지: Snort Alert 발생

Scenario 3: Command Injection Attack (/etc/passwd)

공격: 웹 파라미터를 통한 시스템 파일 접근 시도
탐지: "etc/passwd" 패턴 매칭 탐지

Scenario 4: DDoS - Random Source Attack

공격: Kali에서 Random Source DDoS 수행
탐지: 비정상 트래픽 플러딩 탐지

Scenario 5: SYN Flag Scanning

공격: 포트 스캐닝 (SYN Flag)
탐지: 스캔 행위 식별

05. Threat Detection Scenarios (Suricata Rules)

멀티스레드 기반의 IDS인 Suricata를 활용하여 동일한 공격 시나리오에 대한 탐지 성능을 교차 검증했습니다.

Scenario 1: ICMP & HTTP Traffic

공격: Ping 트래픽 발생
탐지: fast.log 확인 (ICMP)
HTTP 트래픽 탐지 로그

Scenario 2: Inter-Network Ping (Rocky -> Win10)

공격: Ping 전송
탐지: Suricata Alert 확인

Scenario 3: Command Injection Defense

공격: 악성 페이로드 전송
탐지: 시그니처 기반 공격 탐지

Scenario 4: DDoS Detection

공격: Random Source Flooding
탐지: 임계치 기반 탐지 로그

Scenario 5: SYN Scan Detection

공격: Nmap SYN Scan
탐지: 스캐닝 패턴 탐지
